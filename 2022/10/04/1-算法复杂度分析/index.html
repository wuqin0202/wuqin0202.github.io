1.算法复杂度分析

一、渐进符号

（一）、\(O\) 符号

  1. 定义：\(\exists\) 自然数 \(n_0\) 和常数 \(c>0\) ，使 \(\forall n \geq n_0\) , \(f(n) \leq cg(n)\)

  2. 结论：\(\lim\limits_{n \to \infty} \frac{f(n)}{g(n)} \neq \infty \Rightarrow f(n) = O(g(n))\)

    意味着 \(g(n)\) 的增速不能小于 \(f(n)\) ，之所以不是\(\lim\limits_{n \to \infty} \frac{f(n)}{g(n)} = 0\) 的形式是因为 \(g(n)\) 的增速可以等于\(f(n)\)

（二）、\(\Omega\) 符号

  1. 定义：\(\exists\) 自然数 \(n_0\) 和常数 \(c>0\) ，使 \(\forall n \geq n_0\) , \(f(n) \geq cg(n)\)

  2. 结论：\(\lim\limits_{n \to \infty} \frac{f(n)}{g(n)} \neq 0 \Rightarrow f(n) = \Omega(g(n))\)

    意味着 \(f(n)\) 的增速不能小于 \(g(n)\) ，之所以不是\(\lim\limits_{n \to \infty} \frac{f(n)}{g(n)} = \infty\) 的形式是因为 \(f(n)\) 的增速可以等于 \(g(n)\)

（三）、\(\Theta\) 符号

  1. 定义：\(\exists\) 自然数 \(n_0\) 和常数 \(c_1\) 、\(c_2\) \(>0\) ，使 \(\forall n \geq n_0\) , \(c_1g(n)\leq f(n) \leq c_2g(n)\)
  2. 结论：
    1. \(\lim\limits_{n \to \infty} \frac{f(n)}{g(n)} = c \Rightarrow \Theta(g(n))\) ，其中 \(c\) 为常数
    2. \(f(n) = \Theta(g(n)) \Leftrightarrow f(n) = O(g(n)) \and f(n) = \Omega(g(n))\)

（四）、\(o\) 符号

  1. 定义：\(\forall\) 常数 \(c>0\) ，\(\exists\) 自然数 \(n_0\) ，使 \(\forall n \geq n_0\) , \(f(n) \leq cg(n)\) ，记作 \(f(n) \prec g(n)\)

    与 \(O\) 符号的区别：\(O\) 为 \(\exists\) 常数 \(c\) ，而 \(o\) 为 \(\forall\) 常数 \(c\)

  2. 结论：

    1. \(\lim\limits_{n \to \infty} \frac{f(n)}{g(n)} = 0 \Rightarrow f(n) = o(g(n))\)
    2. \(1 \prec \log \log n \prec \log n \prec \sqrt{n} \prec n^\frac{3}{4} \prec n \prec n \log n \prec n^2 \prec 2^n \prec n! \prec 2^{n^2}\)

（五）、\(\omega\) 符号

  1. 定义：\(\forall\) 常数 \(c>0\) ，\(\exists\) 自然数 \(n_0\) ，使 \(\forall n \geq n_0\) , \(f(n) \geq cg(n)\)
  2. 结论：\(\lim\limits_{n \to \infty} \frac{f(n)}{g(n)} = \infty \Rightarrow f(n) = \omega (g(n))\)

补充：

  1. 对 \(\Theta\) 、\(O\) 、\(\Omega\) 符号的理解
  1. 最优算法：已证明求解一问题的方法的时间复杂度必为 \(\Omega(g(n))\) ，则称在 \(O(g(n))\) 时间内解决该问题的算法为该问题的最优算法

二、时间复杂度分析

（一）、分析对象

  1. 迭代次数：计算循环执行次数
  2. 基本运算次数：计算基本运算的次数
    1. 元运算：加减乘除、比较和逻辑运算、赋值运算
    2. 基本运算：算法中具有最高频率的元运算

（二）、分析方法

  1. 最坏情况分析：分析最坏情况下算法的迭代次数或者基本运算次数

  2. 平均情况分析：分析平均情况下算法的迭代次数或者基本运算次数，须知道每种情况的分布，一般为均匀分布

  3. 递推关系分析：常用于分析递归算法的时间复杂度

  4. 平摊分析

    1. 使用情况：算法运行时间始终变动，大多数时候运行地很快，偶尔花大量时间

    2. 例题 有一空的双向链表 L ，整数数组 A[1···n] 作为输入。若 A[i] 为奇数则将 A[i] 插入双链表 L 中；若 A[i] 为偶数则先将 A[i] 插入双链表中再删除链表 L 前面的奇数元素。

      若用最坏情况分析：最坏情况下为 n-1 个奇数后跟 1 个偶数，则删除次数为 n-1 ，即 while 循环执行次数为 n-1 次，需要 \(O(n)\) 的时间， for 循环执行了 n 次，因此可得算法时间复杂度为 \(O(n^2)\) 。

      此时上界估计过高了

      若用平摊分析：插入次数一定为 n 。每个元素最多删除一次，删除次数为 0 ～ n-1 之间。插入与删除的次数总和为 n ～ 2n-1 之间，因此算法时间复杂度为 \(\Theta(n)\)
